开始使用Fedora

| Comments

离最近一次是使用Fedora Core系列发行版还是3年半前，当时使用的Fedora Core 3，因为工作的原因，放弃了Fedora Core 3的继续使用，其后一直在使用RedFlag 系列版本，包括服务器版本，桌面版本和社区版本，不过最近一段时间又放弃了，原因主要是两个：一个是升级的原因，目前RedFlag版本平滑升级的成本很高，而且因为精简的原因，导致安装软件存在复杂的依赖关系，二是想看看其他发型版本的特点。

因此开始使用目前人气很高的Ubuntu，之所以用Ubuntu，是因为它基于Debian，其架构和RedHat,RedFlag都不相同，想看看这种架构有什么优点。 从Ubuntu 7.04到Ubuntu 8.10，恕我愚钝，没有看出两者架构的优劣之别，不过从实用性，易用性，人性化方面，ubuntu确实比较优秀，其极其丰富的仓库是其易用性的基础，也是其亮点。

可惜，我现在又要放弃ubuntu，不是不够优秀，应该说他完全满足我私人办公的需求，然而在工作方面，却受到了很大的挑战，因为RedFlag和Debian的架构不同，导致在技术支持的时候，无法通过在自己本子上的操作来告知用户也这么做，这样，很多操作就需要回忆以前的操作，这导致支持的效率非常低，而且出错率也很高，长此以往，会让自己的工作不在状态，是有必要改变这种境况了。

Fedora Core 10是最新的Fedora系列版本，它能勾起我的回忆，不过安装完后，很多地方让我失望，平心而论，我暂时没有看到Fedora Core 10比Ubuntu 8.10优秀在哪些地方，也许它的优秀需要慢慢的挖掘吧，就像美女要慢慢欣赏一样？

当然，用上它，对我的工作却是方便了很多，有什么办法呢，为了賺奶分钱，也只好牺牲便利性了。

P.S. 很久没有更新博客了，很很久没有写技术性文档了，这不是个好现象。
