赚奶粉钱第一步--投放广告

自从第一个 google 的广告被枪毙了后，就一直没有信息再做 blog 广告了。上次可以是到 75 美元被 kill 的呀，想来都心疼。

今天和 lancy 在计划明年的财务，看来要达到预期目标，还是有很大的困难，既要开源，也要节流，当然更多的应该是节流。因此一切可以赚钱的机会都应该利用起来，唉，结婚了，就是这么现实。

于是考虑重新拾起网站广告，管他有没有收入，不放广告，肯定没有，放了至少还有点希望吧。

google 的广告是没有办法再放了，baidu 的呢，一直都没有兴趣，于是考虑投放阿里妈妈的广告，这是新的广告交易平台，值得尝试。

广告总是讨厌的东西，所以不希望看到广告的人，尽可以使用广告屏蔽插件来屏蔽，当然，还是要看我的 blog 闲得慌的时候，点击一下广告，我会更加感谢。

暂时投放两个位置，一个是右侧栏，另外一个文章页头。右侧栏是图片 /FLASH 广告，文章页头是文字链接广告。

为了防止一些不堪入目的广告出现，我在选择广告主的时候，仅选择了科技类和网站建设类，希望不会出现比较恶心的广告。

11月 6 2007

PhysicalDrive0

在 Linux 里使用 qemu，当需要从本地实际硬盘启动和运行 OS 的时候，只要给出-hda /dev/hda这样的参数就可以了。
因此 Linux 要表示某一个硬盘，实在是很简单，虽然很多 Linux 初学者觉得 /dev/hda 这样的名字怪怪的。
而在 Windows 下，想表示某一个硬盘，可不是那么简单，表示某一个分区也没有那么容易。我最开始在 Windows 下使用 qemu 要运行我本地硬盘的 linux 的时候，我就找了很久的资料来确定如何表示一个硬盘。
如果做过 windows 驱动程序开发的人，估计就不用我这么累的去找了。
今天又要用 qemu 临时启动我的 DC5.0，结果又把这个参数忘记了，费了半天劲回忆起来了，于是决定写到这里。也就是我标题写的了。
\\.\PhysicalDrive0表示第一个硬盘，以此类推，\\.\PhysicalDrive1是第二个硬盘。 不过不知道 windows 下是否区分 IDE 硬盘和 SCSI/SATA/SAS 硬盘的区别，应该不需要吧。哎，还是对 Windows 了解太少了。
再一次诅咒我的 CDMA，害死人呀！

IBM DS3200存储双控制器变单控制步骤

案情：

最开始用的是两个控制器，每一个控制器一个主机端口。每个端口分别连接一台服务器，准备构成 HA 环境，但是却发现，链接 A 控制器的服务器并不能看到属于 B 控制器管理的 LUN。那这就是一个大麻烦，询问 IBM 800，已经确认这种情况下是做不到一个 LUN 被两台服务器同时看到的。
因此决定去掉一个控制器，然后在剩下的控制器上，增加一个 SAS 2-port Daughter Card 39R6509 的子卡, 以达到有 3 个主机端口的目的，从而做到 HA 环境。
