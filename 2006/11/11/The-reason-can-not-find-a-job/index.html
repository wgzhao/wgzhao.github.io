11月 11 2006

找不到工作的原因

又翻了十几页招聘信息，勉强发了两份简历出去。
通过这么长时间零星的找工作中，我大概可以得出我找不到工作的原因：

1）我有技术支持工作经验，对不起，我们不要 Linux 技术支持的人才，我要精通 windows 服务器版本的人才；

2）我也算有 web 开发经验，对 LAMP 的开发也比较熟悉，对不起，我们只要精通 ASP，JSP 的人才；

3）编程吧，写 C 语言也能硬着头皮上，对不起，我们只要精通.NET、JAVA 平台的人才；

4）去培训教书我也有经验，好歹还要如假包换的高等学院计算机专业教师证，对不起，我们只要精通.NET 的人才。

怎么一路下来，几乎都成了 MS 培训工厂呀，从学校到公司。国家不是天天喊要支持国产软件，振兴民族企业，推广 Linux 不？

一方面媒体大势渲染 Linux 人才如何重要，Linux 人才如何宝贵，Linux 人才如何缺少，可是，我们放眼望去，真正要 Linux 的企业有多少？

Rachel说如果她离开现在的公司，她不知道能干什么，不知道还能找到一份工作？
一旦你想学习 Linux，想真的进入这个行业，你就发现，你进入了一条很窄的道路，你无法看到更多的东西，你就只能在这个狭窄的天空中生活。

为什么？

目前的人才培养是源头，在高校，计算机专业的学生 4 年下来，可能不如一个在培训学校培训一年的人。企业招聘人更多的是想如何让你尽快的能工作，能为其创出价值，而不是如何培养你。于是精通这个，精通那个的要求就出来了。
于是迫于压力，大学生们上课的少，每天抱着这本编程书，那本编程书看，刚看完这本，又出新版本，又学习新版本。而很少有时间去看看计算机专业中非常重要的算法，数据结构，应该说还有高等数学，离散数学等。
于是我们看到，在学校，学习成绩优异的人找工作就比不过那些成天看编程书籍的人。于是他们更多的选择考研，在国内读研，好比刚出狼群，又如虎口。
研究生，什么是研究生，应该是属于在本科打好了基础，在继续深入研究，做科学探讨的过程，可是我们看看大部分那些可怜的研究生们，一个个几乎都成了廉价劳动力甚至免费劳动力，还美其名曰：“培养自己的动手能力”，我看做熟练工种
。
很多研究生 2，3 年下来，导师都难得见上几面，试问，这样的情势下，如何做学问，如何做指导？

所以现在有更多的导师自己开公司，为什么，有廉价的劳动力呀。我并不反对导师开公司，恰恰想法，我赞同这种模式，他为他的学生带来的实际操作的机会。但是这有一个度的问题。不是把你一天到晚抓到公司上班。

我一直都希望自己能读书，但是不想考研，听了，看了太多人的经历，那些研究生们和在公司上班有什么区别？

所以有人提出，中国的研究生教育最失败的教育，研究生教育本科化，博士教育研究生化。有的时候我真不知道教育部得出的中国教育是成果的理由从哪里来？这和当年说医疗改革基本成功有什么区别？

唠唠叨叨，又愤青了一回，该洗洗睡了。
