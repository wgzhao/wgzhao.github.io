PhysicalDrive0

| Comments

在 Linux 里使用 qemu，当需要从本地实际硬盘启动和运行 OS 的时候，只要给出-hda /dev/hda这样的参数就可以了。因此 Linux 要表示某一个硬盘，实在是很简单，虽然很多 Linux 初学者觉得/dev/hda 这样的名字怪怪的。而在 Windows 下，想表示某一个硬盘，可不是那么简单，表示某一个分区也没有那么容易。我最开始在 Windows 下使用 qemu 要运行我本地硬盘的 linux 的时候，我就找了很久的资料来确定如何表示一个硬盘。如果做过 windows 驱动程序开发的人，估计就不用我这么累的去找了。今天又要用 qemu 临时启动我的 DC5.0，结果又把这个参数忘记了，费了半天劲回忆起来了，于是决定写到这里。也就是我标题写的了。
\\.\PhysicalDrive0表示第一个硬盘，以此类推，\\.\PhysicalDrive1是第二个硬盘。 不过不知道 windows 下是否区分 IDE 硬盘和 SCSI/SATA/SAS 硬盘的区别，应该不需要吧。哎，还是对 Windows 了解太少了。再一次诅咒我的 CDMA，害死人呀！
