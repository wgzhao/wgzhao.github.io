第一次真正尝试到NFS的好处

| Comments

以前也配置和使用过NFS,不过那是为了学习如何使用,没去真正去用他。倒是samba用了不少，为了windows的cifs通信。
前几天一直在 windows下使用ultraedit远程修改公司的网站。
因为ultraedit支持直接从ftp打开文件。这样就不用每次都把文件下载下来，修改后再去上传了。

可是的windows终于到了寿终正寝的时候了。在弹出一个“windows到了需要重新安装的时候了，请保存您的所有信心，把含有 Windows xp的光盘插入软驱，点击确定按钮，重启计算机”后，我便转入了Linux。(其实，没有到Linux写程序的主要原因是因为我的Linux不能驱动讯驰技术的无线网卡，不过现在我搞定了，可以参考我写的相应的帖子)。
在google里面搜索了一下linux下面的编辑工具。
看来看去还是觉得anjuta 最好。其它的要不就是功能太单纯，要不就是收费软件。
不过anjuta也不支持从ftp打开文件模式，郁闷！
突然想到我为什么不用NFS呢？公司的服务器我装的式FC2。
应该没有问题。先ssh到公司服务器。 编辑/etc/exports文件。

/var/www/html xxx.xxx.xxx.xxx(rw,anonuid=48,sync)

(注：这里xxx是指我的机器的IP地址)起动对应的服务

#/etc/init.d/portmap start
#/etc/init.d/nfslock start
#/etc/init.d/nfs start

ok，服务端弄好了。
转到我的机器，首先修改/etc/hosts文件，加入下面一行

注yyy是公司服务器地址，后面是公司服务器名称

yyy.yyy.yyy.yyy company-name

保存后，使用root权限执行下面的命令

#mount -t nfs company-name:/var/www/html /jsq_web_nfs

如果没有错误发生，执行df命令你就会看到类似下面这样一行：

company-name:/var/www/html 10321184 225632 9571264 3% /jsq_web_nfs

ok了，现在我就可以在本地修改公司的网站了。

HAPPY！看来在家上班，还是需要做很多前期工作呀！
