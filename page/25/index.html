申请google Adsense不容易

| Comments

最早申请 Google adsense是在 2005 年的 11 月，当时给出的拒绝是资料填写不完整或者违反相关政策，我想决绝的理由应该是后者了，应该是访问量达不到吧。后来访问了多了一些的时候，我又一次申请 Google Adsense，时间是 2006 年的 1 月底，这次没有拒绝我，只是给了一个石沉大海的回音。

两次打击后，几乎没有再次申请 Google Adsense 的欲望了。但是……

上次 brisy 来我这里，要我帮他申请了 adsense，我向他描述了我申请的惨痛经历，也随便填写了一些资料。 4 天后，brisy 告诉我他的申请通过了，而且目前已经有收入了。这对我算是不小的刺激吧。他的访问量没有我高，但是 PR 值就比我高多了，但是我也看到很多没有访问量也没有 PR 值的 blog，怎么能申请到 adsense 呢？

还是再申请一次吧，这次不填写我的资料，填写 lancy，她有固定地址，固定电话等，我现在是什么都没有，填写她的应该靠得住一些。事后证明这个决策是英明的，4 天后，我的 google 申请也通过了，折腾了半天在我的 blog 上和论坛上都投放了一些广告（大家不要给我面子，只管点击，呵呵）。效果嘛，目前还没有瞧见。用 brisy 的话说是“觉得心里踏实一些”。我到没有这个想法，管他呢，先投着吧。

精通意味着什么

| Comments

上午继续我的 it732 网站开发。没有太多进展，自顶向下设计方案和自底向上设计方案同时在用，因此有时两个方案没有走在同一条路上，设计和开发一个人还真累。

下午去面试了一家培训机构，也许是我的简历做的不够吧，没有太多“精通”的东西，导致别人怀疑我对 Linux 不熟悉。希望我能多用命令行工作。而且他们的要求也很高，大部分的东西都要精通，他们定义的精通是了解所有，了解每一个细节，比如你需要了解所有 cisco 产品，并精通它。或许真能找到这样的人吧，至少在他们培训机构的现有人员应该是达到了这个水平（否则怎么招聘你来？） <!-more—> 接着带我参观了他们的培训教室和机房，网络机房里刚好有两个学生在练习，面试的人对我说，你看看这些系统你用过吗？这句话很有诱惑力，虽然在门外我已经知道他们在用什么系统，但是我还是进去了，一个人在用着 windows 2000，另外一个人也在用着 windows 2000，只是用虚拟机跑了一个 RedHat 9.0，庆幸我能说出这两个系统的名字，更应该庆祝我能说出 linux 系统的内核版本。不过人家纠正我说，那不是真正的 linux，那是用虚拟机运行的，我做恍然大悟状，哦，那 qemu 似乎也能做这个吧。

回头看我的简历，只写了”精通 Linux 操作系统及其各种服务配置”，写的时候诚惶诚恐，精通，在我的眼里，那是渴望不可及的一个境界，我知道我对 Linux 的认识才开始。没有在 Linux 下做过太多的开发，没有认真阅读过 Linux 内核代码，顶多算是 Linux 的爱好者和折腾者，精通者，远在天边。

或许长沙对人才的要求太高，还是北京对人才的要求太低，我不理解。

正式面试前我有十几分钟等待时间，旁听了几分钟在机房的课程，应该是讲网络应用吧（似乎 BENT 没有这个课），听到了精通~络的讲师这么一段，大意是这样：

“网页有静态和动态…….其实动态的网页基本上从网页文件名后缀可以看出来，比如 asp 结尾的，比如 php，比如 jsp，当然还有，比如 cgi 结尾的， 比如 xhtml，比如 shtml，比如 XML…..”

看来果然是精通网络人士了，XML 要是有交互功能，那世界将会变成怎样？

走出这个培训机构我在想，我是为自己的能力不足汗颜，还是应该考虑别的。

晚上和网讯的另外一个股东交换了意见，给他看了一些优秀的 web2.0 的网站和他们的现有以及潜在价值，当然包括连今天《新闻联播》都报道的 Google 收购 youtube。我的目的是要让相信，并不总是满屏显示的首页超过看上去简陋的首页，都会说 google 的首页简单~但没有人敢说他的功能简单（也许百度的 CTO 会这样说吧）。

定义的标准应该是用户的体验。说服了他们，我的首页就容易得多了。

忙的时候总是容易忘记重要的事情，昨天还记得是今天是姐姐的生日，到了今天就差点忘记，打电话不在。只好发了短信，生日快乐！
