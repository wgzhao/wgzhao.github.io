Device-mapper-multipath的灵异事件–3

| Comments

来深圳出差是解决 openpower710+redflag 连接 EMC CX300 存储的故障问题，而这个连接方案我已经在北京作的测试，也有对应的测试报告，但是同样的环境，在深圳却找不到存储设备，很是怪异。而最后的解决办法更是让我搞不明白，因此标题冠以“灵异”二字，而具体的情况估计也说不太清楚，就贴上我的当时的一些记录吧，这是其三

将前两天配置的机器重新安装，重新调试 multipath，数据读写都没有任何问题，估计昨天出错的原因是原来的 VG 是在一个控制器使用下的情况配置的，导致现在双控制器下读写不正常。但是 multipath 却出现了路径不能”愈合”的现象，既拔掉一根光纤线后，一个路径失效(假设 sde),然后重新介入，发现该路径依然无效。同时使用 multipath -v3 时还是会给出下面的错误：

path not correctly configured for failover

该信息很难在网络上找到对应的资源。而 multipathd 服务也会在路径失效的情况下异常退出。而该 daemon 程序就是用来检查路径状态和恢复链路的服务程序。重启 multipathd 服务后，会给出下面的错误：

device-mapper: dm-multipath: Failing path 8:48.
device-mapper: dm-multipath: Failing path 8:80.
device-mapper: dm-multipath: Failing path 8:32.
device-mapper: dm-multipath: Failing path 8:64.

从这个错误信息来看，multipathd 并不认为路径状态是好的，虽然你可以正常读写每一条路径。因此使用 multipath -ll 看到下面的信息

[root@SZFIMSDB ~]# multipath -ll
3600601609f70140076916a71f81edb11
[size=66 GB][features="1 queue_if_no_path"][hwhandler="1 emc"]
\_round-robin 0 [enabled]
\_9:0:0:0 sdc 8:32 [faulty][active]
\_ 10:0:0:0 sde 8:64 [faulty][active]
\_ round-robin 0 [enabled]
\_ 9:0:1:0 sdd 8:48 [faulty][active]
\_ 10:0:1:0 sdf 8:80 [faulty][active]

后面的[faulty]是表示路径的状态，faulty 表示当前无法处理 I／0 请求。这应该是不能路径不能愈合的原因。无数电话和研发沟通后，研发认为是工具在误报，可以考虑换一个 multipath 版本测试或者重新编译新的 multipath。

测试了 DC5.0 sp1 自带的 0.4.5-12 的版本和研发先前给我的 0.4.5-6 的版本，还是有这个错误。修改过无数种 multipath 的配置，仍然无效。

Grub提示符下安装grub

| Comments

一直在找如何在 grub 丢失，只剩下 grub>的情况下，直接安装 grub 菜单，而不是输入 root,kernel,initrd 一堆命令后进入系统，再使用 grub-install 来安装 grub。今天 Rachel 从她的笔记本上找到了这条命令，果然管用，记录如下： grub>install (hd0,4)/boot/grub/stage1 (hd0) (hd0,4)/boot/grub/stage2 p (hd0,4)/boot/grub/grub.conf 注意：这里表示根文件系统是 hda5，同时 boot 没有单独分出来。如果你的 boot 是单独分出来的，那么“(hd0,4)/boot”两处就要修改成你的 boot 分区了。

用户级文件系统－－FUSE

| Comments

FUSE 是定义在用户空间(userspace)一组实现文件系统功能的程序，他包括的功能有：

  1. 简单 API 实现
  2. 简单安装
  3. 用户空间
  4. 非特权用户可以使用
  5. kernel 2.4 和 2.6 均可以使用
  6. 目前比较稳定

其实 FUSE 只是一个框架（framework），我比较喜欢的是基于它的各种文集系统的实现，比如 bluetooth filesystem
ghoto2 file system
ssh filesystem
flickr filesystem
blog filesystem
gmail filesystem 等。

我测试了一下ssh filesystem，觉得不错。下面是我的安装和测试手记

1）下载最新的 fuse 版本

2）编译安装

 $tar -vxzf fuse-2.5.3.tar.gz 
 $cd fuse-2.5.3 $ 
 ./configure --with-kernel=/usr/src/linux 
 $make && sudo make install 
 $sudo ldconfig

3）下载最新的 sshfs

4）编译安装

 $tar -vxzf ssfs-fuse-1.6.tar.gz 
 $cd sshfs-fuse-1.6 
 $./configure && make && make instll

5）测试

你可以使用通常的挂载分区的方式挂载远程服务器上的某一个目录，而通过的协议是 ssh，比如

#sshfs mlsx@172.16.81.196:/home/mlsx /misc
[root@wgzhao ~]# df -h 
Filesystem 容量 已用 可用 已用% 挂载点 
/dev/hda5 5.8G 4.6G    987M 83% / 
none 248M 0 248M 0% /dev/shm 
/dev/hda6 17G 4.1G 13G 25% /home
/dev/hda7 7.9G 4.5G 3.0G 61% /mnt/suse
sshfs#mlsx@172.16.81.196:/home/mlsx 7.5T 0 7.5T 0% /misc

不过它对大小的识别就有点问题了。不管是哪个分区，哪个目录挂载上来，它都识别为 7.5T（如果是真的，那该多好）。这也挂载上来的读写权限是远程机器帐号的 uid 和 gid 对应的本地机器的帐号，比如如果远程机器 mlsx 帐号的 uid 是 500,而我的机器上 uid 是 500 的帐号是 work，那么/misc 的读写权限属于 work，其他帐号都没有权限。你可以这样来挂载 #sshfs mlsx@172.16.81.196:/home/mlsx /misc -o uid=1000,gid=100 uid，gid 是对应你本地机器帐号的 id 号，这样你就可以指定某个帐号来读写了。卸载的时候，使用下面的命令 #fusermount -u /misc 是不是觉得比 scp，sftp 要方便很多呢？同样的，你可以使用它提供的 gmail filesystem，blog filesystem 等。而接下来，我要关心的是基于 fuse 的 ZFS，这是 sun 公司基于 solaris 提出的一种文件系统，在我的 blog 中提到过它，而 ZFS-FUSE 就是基于 FUSE 来在 linux 实现 ZFS 的特性，不过目前还没有实质性的成果，我也下载了它现在推出的最新版本，也能编译通过，但是目前还不知道如何使用，solaris 上有专门的视频来告诉你如何使用 ZFS，可惜那些功能目前在 ZFS-FUSE 还没有实现。让我们期待。

Novell SuSe 10.0使用感受

| Comments

前两天同事下载了 Novell SuSe Enterprise Desktop 10.0 ,简称 SLED10。总算是看到了 xgl 带来的 3D 绚丽桌面效果。就像下面的截图一样

如果是以前，我肯定毫不犹豫的安装了。但是 3 年前疲于各种系统安装尝鲜的我已经没有了当年的激情。尝鲜已经不是我首要的任务，我需要的是稳定的桌面，可用的办公系统等。不过基于我有喜爱的 gnome，和它自带的搜索引擎。经过一天的考虑，我还是决定安装。

我的机器是 HP compaq nx6120，迅驰 1.7GHz/512M/intel915。应该还算可以吧。安装大约是 40 分钟。安装下来 4G。默认启动 gnome，3D 桌面默认没有打开，你需要在控制面版的桌面效果中启动 3D 桌面效果。于是我看到了旋转的 3D 桌面，多达 36 个桌面形成的多面柱体。也看到了窗口颤抖效果，水波纹效果。跑起来还算流畅。自带的软件果然是很多，全新的菜单方式也有耳目一新的感觉，不过因为安装的软件过多，加载菜单时感觉很慢。显然 3D 效果并不能为我带来多大生产力，我需要的是可用的办公桌面。首先看看 office 如何，自带的是 novell 经过修改的 openoffice2.0，当然，我不知道修改了什么，可能是兼容性问题吧。不过和 ms 的 office 兼容还是存在问题。不得不安装永中 office 2007，安装一切正常。但是打开文档会崩溃，在终端留下遗言，是 java 的问题。修改配置，使用 suse 自带的 java，也不行。安装 jre1.5，也不行，依然是崩溃。这个先放到一边，实在不行还可以使用 redoffice,openoffice。再看看 IM，启动 gaim，默认是 1.5 版本。登录 msn 特别慢。慢也就算了，能登录就行，问题是它时不时崩溃，这就让人很难接受了。也许是版本的问题，到官方站点下载了 2.0 的 beta 版。问题依然。换一个软件，使用 kopete，登录比 gaim 快一点，使用了不上时间，直接报 KDE 崩溃。重新打开，问题依然。不过没有尝试过在 KDE 环境下会如何。打开 firefox，默认版本是 1.5.0.4，最新的稳定版本了。访问网站很慢，主要在解析这块，禁止了 ipv6。还是觉得慢。没有想到其他办法了。运行一段时间后，发现系统明显比使用红旗 5 慢很多。top 一下，发现负载是 2.7，比较恐怖，查看进程，X 经常占用 97%，beagle 也是消耗大户，查看改进程的来源，原来是 novell 的索引引擎。类似 google 桌面，但是这样的速度实在是太慢了点。关闭 3D 桌面，关闭索引引擎，重启桌面，速度快了不少，或者恢复到我能接受的程度。如此以来，我最需要的功能－office，IM 没有办法使用。那这个系统再怎么豪华，对我来说也是白搭，毕竟我不能拿绚丽的桌面工作和赚钱。所以用了不到一天，又重新回到了红旗桌面，虽然红旗桌面有很多我不喜欢的地方，但是他至少能满足我的基本需求。改天再测试它的 kde 桌面吧。如果两个 X－windows 都没有满足我的基本需求，那么我就只能废了他了。
