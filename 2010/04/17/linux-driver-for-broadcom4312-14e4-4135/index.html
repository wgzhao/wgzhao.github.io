Linux下安装Broadcom4312 802.11b/g(14e4:4315)驱动

| Comments

还是没有忍住，提前把系统升级到了Ubuntu 10.04 Lucid LTS版本。 升级后，需要把原来用户的$HOME/.gconf删除一些桌面的配置，主题等。
因为10.04用了全新的主题，如果不删除，看到的就还是老的9.10的样子了. 目前除了主题有所变化外，暂时还是没有看出什么区别。
我急着升级是因为linuxwireless 上说我的无线网卡(14e4:4315)在2.6.32内核已经支持支持了。

But，我没有发现这点。我记得我在9.10下搞定无线网络，最终用的是一个wl的驱动。但是10.04的核心还是没有看到这个驱动。

尝试了linuxwireless上面说的几个办法，还是没有搞定。于是去broadcom 官方站点看了看，还是官方牛逼，下载他的驱动就搞定了。记录如下：

  1. 从官方站点，下载对应架构的驱动，地址是这里 http://www.broadcom.com/support/802.11/linux_sta.php
  2. 解压，编译，安装。这个是老套路，应该不会有报错。
  3. 执行depmod -a 上一个步骤里make install 居然没有执行depmod -a

  4. 屏蔽掉系统自带的驱动

    echo "blacklist b43 " >>/etc/modprobe.d/blacklist.conf
echo "blacklist ssb" >>/etc/modprobe.d/blacklist.conf

  5. 在/etc/modules加入wl模块 echo "wl" >>/etc/modules

  6. 重做引导镜像 mkinitramfs -o /boot/initrd.img-2.6.32-21-generic `uname -r`
  7. 重启
  8. 如果没有驱动起来，看看是不是没有安装firmware固件。 sudo apt-get install b43-fwcutter

如果觉得我说得太简单，下面的这篇Wiki更详细，不过准备要梯子
