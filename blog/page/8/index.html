E13.1 研究Linux的route命令，简要介绍它的功能。使用route命令，怎样做到：

  1. 加一条通过接口 eth1 到 128.138.129.1 的默认路由；
  2. 删除到 128.138.129.1 的默认路由；
  3. 判断一个程序 (routed)或者一个ICMP重定向是否加入了一条路由（注意，这个方法也能用netstat -rn 命令的输出做到）。

A：route的主要目的是现实和修改内核IP路由表信息。参数比较多。

  1. route add default gw 128.138.129.1 eth1
  2. route default default gw 128.138.129.1
  3. 我的网络环境没有ICMP重定向，所以也看不到是否增加了路由信息。

E12.2 网络链路的MTU是什么？如果给定的MTU设得太高，会发生什么问题？太低呢？

A：MTU（Maximum Transfer Unit)，最大传输单元，用来定义网络上传输包的最大值。以太网一般是1500bytes。太高会导致丢包，太低则导致分片，都会产生网络传输问题。

Linux系统管理技术手册第10章系统实践

| Comments

E10.1 保留老日志文件的主要理由是什么？

A：除非你每天查看系统日志并记录有故障的消息，否则你就要保留老的日志文件，否则BOSS问起你来，你来什么来对付呢？

E10.2 lastlog和wtmp之间的区别是什么？每种日志合理的轮换策略是什么？

A：lastlog记录的是当前所有系统帐号的登录情况，记录每一个帐号最近的登录日期，登录端口/终端。如果没有登录过，则显示为从未登录。
而wtmp则是记录所有登录帐号的信息，包括该帐号登录的时间，登录终端，退出时间等。

2010年计划

| Comments

2009年开始到来的时候，我做了一个计划。现在是总结的时候了

  1. 好好抚养皮皮：目前皮皮正在健康快乐的成长，到目前为止，暂时还没有感冒过，发烧过，除了打预防针，暂时还没有去过医院，这一点让我们觉得皮皮真的很好带，也让我们省了很多心。同时也觉得小孩的东西，质量还是很重要的，这一点，我打算到时专门写一篇blog，谈谈我对购买婴儿用品的看法和理解。
  2. 米米当然还是很重要的：唉，金融危机呀，到目前为止，属于深度套牢，基本上没有增加结构性收入，创造性收入也非常少。
  3. 如果金融危机不再加深，考虑在这年买房：这个做到了，4月底买的房，目前正在装修，如果没有特别情况发生，应该可以在农历新年之前完工，不过今年不打算入住。

  4. 技能方面：

    • 数据库技术方面，想重点学习Oracle和MySQL：买了一本Oracle的书，也系统的学习了下，还是有明显进步的。MySQL没有完成，再一次发现，还是得买印刷的书，电子书容易走神。
    • 集群和虚拟化技术： 作了虚拟化的一个专题研究，对heartbeat也有了一些研究，但是都比较肤浅，这点完成得不是很理想。
    • 系统管理技术：看了一些电子类的书籍，年终的时候，还是决定买下《linux管理技术手册》一书，并打算完成所有章节后的习题，这算是我系统学习/sbin,/usr/sbin后的另外一个长期实践吧。至于学习python，开了个头，还是没有找到很好的深入学习方式。

  5. 增加知识面，就需要增加自己的阅读量、阅读面：2009年，读了不少非专业书籍(在豆瓣网上搞了一个豆列，不包括电子书），可以说绝大部分是买的非专业书籍，对增加自己的知识面和量都大有裨益，需要继续坚持。

—————————2009年的分割线——————————–

说到2010年的计划，除了2009年的一些计划可以延续外，2009年的最后几天增加了一个突发因素，如果这个突发因素导致的事情最后实现的话，那么2010年会有一个相当大的变化，所以其他一些计划目前就不好制定了，一切都要等把这个突发因素带来的影响消化完。

等着吧。
